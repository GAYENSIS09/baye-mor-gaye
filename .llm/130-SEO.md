# CONTRAT OPTIMISATION POUR LES MOTEURS DE RECHERCHE (SEO)

## 1. STRATÉGIE GLOBALE SEO TECHNIQUE

Le référencement naturel (SEO) de notre application repose sur une fondation technique solide. Chaque aspect du développement, tant côté backend (Laravel) que frontend (Next.js), doit être pensé pour favoriser l'indexation, la compréhension et le classement par les moteurs de recherche (Google, Bing, etc.).

---

## 2. EXIGENCES TECHNIQUES POUR LE SEO

### 2.1 Structure Sémantique du Contenu (HTML5)
*   Utiliser les balises HTML5 sémantiques (`<main>`, `<nav>`, `<h1>`-`<h6>`, `<article>`, `<aside>`) pour structurer le contenu de manière hiérarchique et logique.
*   S'assurer que le contenu principal d'une page est contenu dans la balise `<main>`.
*   Utiliser des titres (`<h1>` à `<h6>`) de manière séquentielle et logique, avec un seul `<h1>` par page pour le titre principal.

### 2.2 Méta-Données Optimisées
*   **Balise `<title>` :** Unique, descriptive et pertinente pour chaque page. Inclure le nom du site/application en fin de titre (ex: "Titre de la Page | Nom de l'App"). Doit être dynamique et généré par Next.js.
*   **Balise `<meta name="description">` :** Unique, concise (environ 150-160 caractères) et engageante pour chaque page. Elle sert de résumé dans les résultats de recherche.
*   **Balise `robots` :** Contrôler l'indexation et le suivi des liens. Utiliser `index, follow` par défaut, et `noindex, nofollow` pour les pages privées ou non pertinentes pour le référencement (ex: pages d'administration, pages de profil non publiques).

### 2.3 Données Structurées (Schema.org / JSON-LD)
*   **Implémentation Obligatoire :** Pour les types de contenu clés (articles, produits, événements, recettes, organisations), intégrer des données structurées au format JSON-LD dans le `<head>` de la page.
*   **Exemple pour un Article :**
    ```html
    <script type="application/ld+json">
    {
      "@context": "https://schema.org",
      "@type": "Article",
      "headline": "Titre de l'Article",
      "author": {
        "@type": "Person",
        "name": "Nom de l'Auteur"
      },
      "datePublished": "2026-01-01",
      "dateModified": "2026-01-01",
      "description": "Description courte de l'article.",
      "image": "URL de l'image principale"
    }
    </script>
    ```

### 2.4 Optimisation des Performances (Crucial pour le SEO)
*   Le respect des métriques LCP, CLS, FID/INP est directement lié au classement SEO. Les mesures prises dans le fichier `100-PERFORMANCE.md` sont donc intrinsèquement liées au SEO.
*   **Vitesse de chargement :** Images optimisées (`next/image`), chargement paresseux (lazy loading), code split (`next/dynamic`), polices locales (`next/font`).
*   **Mobile-First :** Assurer une expérience utilisateur impeccable sur mobile, car Google utilise l'indexation mobile-first.

### 2.5 Structure des Liens Internes
*   Créer des liens clairs et logiques entre les pages pertinentes de l'application. Cela aide les moteurs de recherche à découvrir le contenu et les utilisateurs à naviguer.
*   Utiliser des ancres textuelles descriptives (ex: `<a href="/posts">Voir tous nos articles</a>` plutôt que `Cliquez ici`).

### 2.6 Canonicalisation
*   Pour les contenus potentiellement accessibles via plusieurs URLs (ex: avec/sans paramètres de tri, avec/sans slash final), spécifier la version canonique (`<link rel="canonical" href="...">`) pour éviter le contenu dupliqué aux yeux des moteurs de recherche.

---

## 3. OUTILS ET VÉRIFICATIONS SEO

*   **Google Search Console / Bing Webmaster Tools :** Soumettre le sitemap XML de l'application et surveiller les erreurs d'indexation, les problèmes de performance, et les rapports de sécurité.
*   **Outils de Test SEO :** Utiliser des outils comme Google PageSpeed Insights, GTmetrix, ou SEMrush pour analyser la conformité technique et les opportunités d'amélioration.
